System Status

SeasonCast infrastructure is built with redundancy in mind. The system is designed to be 100% functional with 3/4 ingest locations failing, 2/3 API locations failing. We use a combination of load balancing and failover to ensure that the system is always available. We use different locations and server providers to prevent one source of failover. While ingest locations may & do fail, the SeasonCast app will always failover to different locations regardless of users location. This means a complete outage of SeasonCast would require ether all ingest locations to fail or all API locations to fail. We are commited to providing a reliable service to all of our users.
